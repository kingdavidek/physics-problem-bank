Tech stack & key files:
•	Flask on PythonAnywhere, using a single app.py for routes (including Quick Test routes that pre generate a queue of questions and store them in SQLite).
•	Generators live in generators/<subject>/<module>.py. Each topic has a main function (e.g. gcse_vectors) and a variants function (e.g. gcse_vectors_variants). All generators return a question dict using make_problem() from generators/shared/utils.py.
•	Lesson templates are in templates/ and follow the naming pattern <level>_<subject>_<topic>_lesson.html. They use Jinja2, extend base.html, and use <details> accordions.
•	Topic registry is topic_registry.py – a central dict that maps level/subject/topic to the generator function and an optional variants_func.
•	Frontend uses templates/index.html (the generator page) with dropdowns that are populated manually (options with data-level/data-subject).
•	Quick Test works by calling variants_func to get a list of variant functions, generating all problems, storing them in a SQLite quicktest_sessions table (via _save_qt/_load_qt in app.py), then serving one question at a time.
•	MCQs work by returning a six tuple (question, solution, hint, marks, options_list, correct_letter) and are handled in the main generator with a mode == 'mcq' branch.
•	Math rendering uses MathJax (tex svg) with a configuration file static/js/mathjax-config.js.
•	CSP is handled via WSGI middleware in the WSGI file (strips any existing CSP and injects a permissive one).
•	SVGs are styled globally in base.html so that all instructional SVGs get a visible background and are not treated as MathJax formulas.
