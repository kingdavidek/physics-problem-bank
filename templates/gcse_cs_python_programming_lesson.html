1 Variables, Constants & Data Types
1.1 Variables and assignment

Think of a variable as a labelled box in the computer’s memory: you give it a name, then you put a value inside. While your program runs, you can read that value, change it, or use it in calculations. GCSE exam questions often ask you to trace through code and state what each variable holds after each line — so it helps to picture the “box” updating step by step.

You create or update a variable with the assignment operator = (python understands a single "=" sign as “contains”, and not “equals”). Choose names that describe the data, such as score or player_name, rather than vague names like x unless the question uses them. In the example below, a quiz game starts with score 0, then adds 10 when the player answers correctly.

name = "Alice"
score = 0
score = score + 10   # value can change

1.2 The four data types

Every value in Python has a data type. The type tells the computer how to store the value and which operations are allowed — for example, you can add two integers, but joining two strings uses + to mean “stick together”, not “add mathematically”. At GCSE you need to recognise and use four main types:

int whole numbers · float decimals · str text · bool True/False

Example: storing details for a student — their name is text (str), age is a whole number (int), height might be 1.72 metres (float), and whether they are enrolled is either True or False (bool). Booleans are essential later for if statements and loops.

name = "Alice"        # string (str)
age = 16              # integer (int)
height = 1.72         # float (decimal)
is_student = True     # boolean (bool)

1.3 Checking types with type()

If you are unsure what kind of value a variable holds — especially after input() — you can ask Python with type(x). It returns something like <class 'int'>, which means “this value is an integer”. In exams, tracing programs often hinges on knowing whether you are working with a number or a string; checking the type is the first step when a program behaves oddly (for example, "5" + "5" gives "55", not 10).

Remember: False and True are booleans (no quotes). The text "False" is a string — a common trick in multiple-choice questions.

print(type(42))        # <class 'int'>
print(type("hello"))  # <class 'str'>
print(type(True))     # <class 'bool'>

2 Input & Output
2.1 print() — displaying output

Programs need a way to show results on the screen. In Python, print() displays whatever you put inside the brackets. It is one of the first commands you learn because almost every program uses it — from a simple “Hello, World!” to showing a final score or error message.

You can print several items in one line. Using + only works sensibly when you are joining strings (and you must convert numbers with str() if needed). Using commas is often easier: Python converts each value to text and puts a space between them automatically — so print("Score:", 42) displays Score: 42 without you typing spaces yourself.

print("Hello, World!")
print("Score:", 42)
print("Hello, " + "Alice" + "!")

2.2 input() and type conversion

Interactive programs pause and wait for the user to type something. The input() function shows a prompt (the message in the brackets), reads what the user types, and returns it to your program. That makes programs feel personal — quizzes, login screens, and calculators all rely on input().

Here is the catch that appears in almost every GCSE paper: input() always returns a string, even if the user types digits. So if they enter 17, Python stores "17" (text), not the number 17. Before adding, subtracting, or comparing numerically, convert with int() for whole numbers or float() for decimals.

name = input("Enter your name: ")
print("Hello, " + name + "!")

age = int(input("Enter your age: "))
print("Next year you will be", age + 1)

3 Arithmetic & Comparison Operators
3.1 Arithmetic operators

Arithmetic operators let your program do maths. You will use +, -, and * in the same way as in mathematics. Division / always gives a float (decimal), even when the answer is whole — so 10 / 2 is 5.0, not 5.

Three operators are especially common in GCSE questions: // (integer division — how many whole times one number fits into another), % (modulus — the remainder left over), and ** (power). For example, 17 % 5 is 2 because 17 = 3×5 + 2; modulus is useful for checking even numbers (n % 2 == 0) or cycling through menu options.

a, b = 10, 3
print(a + b)    # 13  addition
print(a - b)    # 7   subtraction
print(a * b)    # 30  multiplication
print(a / b)    # 3.333...  division (float)
print(a // b)   # 3   integer division
print(a % b)    # 1   modulus (remainder)
print(a ** b)   # 1000  power

3.2 Comparison operators

Comparison operators ask a yes/no question about two values. The answer is always a boolean: True if the comparison holds, False if it does not. These operators are the building blocks of if statements — your program branches depending on whether the condition is true.

Students often mix up = and ==. A single = assigns a value to a variable (score = 10). Double == compares two values for equality (score == 10). Using = inside an if by mistake is a syntax error in Python and a frequent exam trap.

3.3 Comparisons in conditions

Comparisons really matter when combined with if. The program below reads an age, converts it to an integer, then decides which message to show. Notice how age >= 18 is a question: “Is age greater than or equal to 18?” — only if the answer is True does Python run the indented block underneath.

Python also allows “between” checks in one line: 1 < x < 10 means x is greater than 1 and less than 10. That is clearer than chaining awkward and expressions and often appears in mark schemes.

age = int(input("Enter your age: "))
if age >= 18:
    print("You are an adult.")
else:
    print("You are not yet an adult.")

Exam pitfall: comparing a string to a number always fails. "10" == 10 is False because the types differ — always use int(input(...)) when you need numeric comparisons.

4 Selection: if / elif / else
4.1 if and else

Selection means your program chooses different paths depending on conditions — like a fork in the road. The simplest form is if … else: one block runs when the test is true, another when it is false. Every GCSE board expects you to read and write these structures accurately.

After if you write a condition (something that evaluates to True or False), then a colon :. The lines that belong to that branch must be indented (usually four spaces). In the quiz example, a score of 65 is at least 50, so “Pass” is printed; otherwise “Fail” would run.

score = 65
if score >= 50:
    print("Pass")
else:
    print("Fail")

4.2 elif — multiple paths

When there are more than two outcomes, use elif (short for “else if”). Python checks conditions from top to bottom and runs the first block whose condition is true. After that, it skips the remaining elif and else branches — even if another condition would also have been true.

Example: with a score of 82, grade >= 90 is false, but grade >= 80 is true, so “Grade: A” prints. The later tests for B and below are never reached. Order matters: put stricter (higher) thresholds first, or a general condition like grade >= 50 might “catch” everyone before more specific grades are tested.

grade = int(input("Enter percentage: "))
if grade >= 90:
    print("Grade: A*")
elif grade >= 80:
    print("Grade: A")
elif grade >= 70:
    print("Grade: B")
else:
    print("Grade: below B")

4.3 Indentation

Unlike some languages, Python does not use curly braces { } to mark blocks. Instead, indentation is part of the syntax: lines that belong to an if, elif, else, loop, or function must be indented further than the line above. If you forget to indent, Python raises an IndentationError; if you indent by mistake, code may run when you did not intend it to.

Use a consistent style — four spaces per level is standard in GCSE mark schemes. The commented “WRONG” line below would run every time because it is not inside the if block. When revising, trace indentation with a pencil: draw a box around each indented block.

if 5 > 3:

4.4 and, or, not

Sometimes one comparison is not enough. You can combine conditions with logical operators: and (both must be true), or (at least one must be true), and not (reverses true to false and vice versa). They are used constantly in validation — for example, checking that a user is old enough and has ID before entry.

Example: a 17-year-old with ID cannot enter because age >= 18 is false, so the whole and expression is false even though has_id is true. With or, only one part needs to be true. Exam trace tables often ask you to evaluate combined expressions step by step.

age = 17
has_id = True
if age >= 18 and has_id:
    print("You may enter.")

4.5 Nested if

A nested if is an if statement inside another if (or else) block. You use this when a decision depends on an earlier decision — first check whether it is raining; only if it is, ask whether it is windy and choose the umbrella type.

Each level needs its own indentation. In the example, if rain is false, the inner if windy is never evaluated and “No umbrella needed.” runs. Nesting is clearer than one long line of and when the logic has natural stages, but too many levels can be hard to read — another elif chain is sometimes better.

rain = True
windy = False
if rain:
    if windy:
        print("Sturdy umbrella!")
    else:
        print("Normal umbrella.")
else:
    print("No umbrella needed.")

5 Iteration: for & while Loops
5.1 for loops and range()

Iteration means repeating code — essential for processing lists, counting, and menus. A for loop runs once for each item in a sequence. The built-in range() function generates a sequence of numbers without you typing them all out.

range(5) produces 0, 1, 2, 3, 4 — it starts at 0 and stops before the number you pass in. That “stop before” rule causes many off-by-one errors in exams: range(1, 11) gives 1 up to 10, not 11. You can also add a third argument for steps, including negative steps for countdowns.

for i in range(5):
    print(i)

for n in range(1, 11):      # 1 to 10
    print(n)

for n in range(10, 0, -1):  # countdown
    print(n)

5.2 Looping through lists and strings

You do not have to use range() every time. A for loop can walk through each item in a list or each character in a string directly. The loop variable (fruit below) takes each value in turn — first "apple", then "banana", then "cherry".

This style is ideal when you care about the items themselves (printing names, totalling scores) rather than the index position. The same pattern works for a username string: for letter in "Python" prints one letter per line. GCSE tasks often ask you to accumulate a total inside such a loop.

fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

5.3 while loops

A while loop repeats as long as its condition stays true. Unlike a for loop, you do not always know in advance how many times it will run — it is the right choice when you repeat until something happens (valid password entered, game still running, counter reached a limit).

Before each repetition, Python checks the condition at the top of the loop. You must include code that eventually makes the condition false; otherwise you get an infinite loop. The counter example adds 1 each time; the password example keeps asking until the user types "secret".

count = 1
while count <= 5:
    print(count)
    count = count + 1

password = ""
while password != "secret":
    password = input("Enter password: ")
print("Access granted.")

5.4 break and continue

Two keywords give you extra control inside loops. break stops the loop immediately — useful when you have found what you were searching for and do not need to check the rest. continue skips the rest of the current repetition and jumps to the next one — handy for ignoring certain values without leaving the loop entirely.

In the first example, as soon as item == 9, “Found!” prints and break ends the loop; 5 is never processed. In the second, even numbers hit continue, so print(i) only runs for odd numbers. Trace these carefully in exams: continue does not exit the whole loop.

for item in [4, 7, 9, 5]:
    if item == 9:
        print("Found!")
        break

for i in range(10):
    if i % 2 == 0:
        continue
    print(i)   # odd numbers only

6 String Manipulation
6.1 String methods

A string is a sequence of characters — names, passwords, messages, or single letters. Python provides methods: functions that belong to a particular string and are called with a dot, such as text.lower(). They do not change the original string; they return a new one (strings are immutable), which is worth remembering when tracing code.

Common GCSE methods include strip() to remove spaces from the ends (useful after input()), lower() and upper() for case-insensitive comparisons, and len() for the number of characters. Validating login details often combines these: strip spaces, then compare with lower() so "Admin" matches "admin".

text = "  Hello, World!  "
print(text.strip())    # removes outer spaces
print(text.lower())    # lowercase
print(text.upper())    # uppercase
print(len(text))       # length

6.2 Indexing and slicing

Each character in a string has a position called an index. Python counts from zero: the first character of "Python" is text[0] which is 'P'. Negative indices count from the end: text[-1] is the last character ('n').

Slicing extracts part of a string: text[0:3] takes characters at index 0, 1, and 2 — it stops before index 3, giving "Pyt". The same rules apply to lists. Exam questions may ask for the substring starting at a given index or the effect of slicing on a word like a password or postcode.

text = "Python"
print(text[0])     # P
print(text[-1])    # n (last character)
print(text[0:3])   # Pyt

7 Lists (Arrays)
7.1 Creating lists and indexing

A list stores several values in order inside square brackets [ ], separated by commas. In GCSE Computer Science, lists are the main way to represent collections — test scores, usernames, prices, or characters in a game. Items can be the same type or mixed, though exam programs usually use one type for clarity.

You access items by index, starting at 0: fruits[0] is "apple". You can also change an item with assignment: fruits[1] = "blueberry" replaces the second element. Using fruits[-1] is a neat way to get the last item without knowing the length.

fruits = ["apple", "banana", "cherry"]
print(fruits[0])    # apple
print(fruits[-1])   # cherry
fruits[1] = "blueberry"

7.2 append, insert, remove, pop

Lists can grow and shrink while your program runs. append(item) adds to the end — like adding an item to the bottom of a shopping list. insert(index, item) places something at a specific position; existing items shift along. remove(value) deletes the first matching value by content, not by index.

pop() removes and returns the last item (useful for a stack of undo actions). Know the difference between removing by value (remove("milk")) and by position — exam pseudocode often says “add to end of list”, which maps to append.

shopping = ["bread", "milk"]
shopping.append("eggs")
shopping.insert(1, "cheese")
shopping.remove("milk")
last = shopping.pop()

7.3 Looping and the in keyword

Combining lists with for loops is a classic GCSE pattern. To find the total of numbers in a list, start a variable at 0, then add each score inside the loop. This is called an accumulator — the variable “accumulates” the running sum.

The keyword in checks membership: "apple" in fruits is True if that value appears anywhere in the list. That is cleaner than looping manually when you only need a yes/no answer. Remember: valid indexes for a list of length 5 are 0 to 4, not 0 to 5 — using index 5 causes an IndexError.

scores = [12, 45, 67, 89, 34]
total = 0
for s in scores:
    total = total + s

7.4 Slicing and 2D lists

List slicing uses the same [start:stop] idea as strings: letters[1:4] returns a new list containing elements at indexes 1, 2, and 3 — here ["b", "c", "d"]. Slicing is useful for taking a subset of results without copying every element by hand.

A 2D list is a list where each item is another list — like a grid or spreadsheet. grid[1][2] means row 1, column 2 (remember both start at 0). In the example, row 1 is [4, 5, 6] and column 2 of that row is 6. Exam questions may show a table of temperatures or a tic-tac-toe board stored this way.

letters = ["a", "b", "c", "d", "e"]
print(letters[1:4])   # ["b", "c", "d"]

grid = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
print(grid[1][2])    # 6

8 Functions & Procedures
8.1 def and calling procedures

As programs grow, repeating the same lines in many places becomes messy. A function (or procedure when it does not return a value) groups code under one name. You define it once with def, then call it whenever you need that behaviour — like pressing a button that runs a saved set of instructions.

The example defines greet() with two print lines indented underneath. Calling greet() twice runs those lines twice. Definitions are not run until you call the function; exam trace questions often jump over the def block until the call line is reached.

def greet():
    print("Hello!")
    print("Welcome.")

greet()
greet()

8.2 Parameters and arguments

Functions become far more useful when they accept parameters — placeholders in the definition for values that can change each time. When you call the function, the actual values you pass in are called arguments.

In def greet_user(name):, name is a parameter. When you write greet_user("Alice"), "Alice" is the argument: inside the function, name temporarily means "Alice". The same function works for any name without rewriting the code — a core idea in structured programming for GCSE.

def greet_user(name):
    print("Hello, " + name + "!")

greet_user("Alice")
greet_user("Bob")

8.3 return values

A function with a return value sends a result back to the line that called it. Use return followed by the value or expression. The caller can store that result in a variable, use it in a condition, or pass it to another function — the function becomes like a small machine that computes an answer.

Example: add(5, 3) returns 8, which is stored in total and then printed. is_even(4) returns True because 4 % 2 == 0. In exams, distinguish return (sends data back) from print() (shows on screen only). If you only print inside a function, the rest of the program cannot use that value in further calculations.

def add(a, b):
    return a + b

total = add(5, 3)
print(total)   # 8

def is_even(number):
    return number % 2 == 0

8.4 Local and global scope

Scope means where a variable name can be used. A variable created inside a function is local: it exists only while that function runs and cannot be seen outside it. A variable created at the top level of the program (not indented under a function) is global and can be read from inside functions unless you create a local with the same name.

In the example, local_name works only inside show(). Trying to print(local_name) after the function finishes causes a NameError — a common exam question. Understanding scope helps you explain why functions are self-contained and reduce accidental changes elsewhere in the program.

name = "global"

def show():
    local_name = "local"
    print(name)       # can read global
    print(local_name)

show()
# print(local_name)  # NameError — local only inside show()

9 File Handling
9.1 Opening files and modes

Programs often need to save data between runs — high scores, user settings, or logs. File handling lets Python read from and write to text files on disk. You open a file with open(), giving a filename and a mode: "r" read only, "w" write (creates a new file or overwrites an existing one), and "a" append (adds to the end without deleting what was already there).

Always use with open("filename", mode) as f: when you can. The with block ensures the file is closed properly when you finish, even if an error occurs — examiners expect this style because it avoids leaving files locked open.

with open("data.txt", "w") as f:
    f.write("Hello\n")

with open("data.txt", "r") as f:
    content = f.read()
    print(content)

9.2 Reading line by line

Large files are usually processed one line at a time rather than loading everything at once. A for loop over the file object reads each line in turn. Text files often end lines with an invisible newline character; line.strip() removes spaces and newlines from the ends so you can compare or print clean text — for example matching a username without extra spaces breaking the comparison.

Example use: a file of quiz scores where each line is one pupil’s result; the loop could convert each line to an integer and add to a total, or count how many lines contain “Pass”.

with open("data.txt", "r") as f:
    for line in f:
        print(line.strip())
