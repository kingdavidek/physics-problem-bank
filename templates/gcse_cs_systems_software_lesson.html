0.2 Why an operating system is needed Year 10 OCRAQA

Without an OS, users would need to talk to hardware in machine code. The operating system sits between applications and hardware, managing resources so programs can share one computer safely.

Platform: Applications are written to run on an OS — Android apps differ from Windows apps.

Worked example: Two pupils print at once — the OS printer queue sends jobs in order instead of corrupting output.

2 Functions of the Operating System Year 10–11 OCR
2.1 Memory management and multitasking Year 10 OCRAQA

Memory management — the OS allocates RAM to each program, frees it when closed, and prevents programs from overwriting each other's memory.

Multitasking — the OS schedules the CPU so many programs appear to run together (time slices). If RAM runs out, the OS may use virtual memory (disk swap) — slower than real RAM.

Worked example: You have a browser, Spotify and Python open — the OS swaps which process uses the CPU thousands of times per second.

2.2 Peripheral management and drivers Year 10 OCR

Peripheral management — the OS controls input/output devices (keyboard, mouse, monitor, USB, printer). It schedules access and handles plug-and-play detection.

A device driver is software that translates OS requests into commands the hardware understands. New hardware often needs a driver installed.

Worked example: You plug in a USB printer — Windows installs a driver so the OS can send print jobs to that model.

2.3 User management Year 10–11 OCR

The OS stores user accounts with usernames and passwords (or biometrics). It enforces permissions — e.g. pupils cannot install software; teachers can.

Groups (Staff, Students) simplify applying the same rights to many accounts.

Worked example: A pupil logs into a network account — the OS loads their profile and only shows their documents, not another pupil's folder.

2.4 File management Year 10 OCR

The OS organises storage using a file system (e.g. NTFS, ext4):

  • Files and folders/directories with names and paths
  • Create, open, read, write, copy, move, delete
  • Permissions — read-only, hidden, who may access
  • Tracks free space and bad sectors (with hardware)

Worked example: Path C:\Users\Alex\work.docx — the OS maps the path to sectors on the SSD via the file system.

3 Utility Software Year 10–11 OCR
3.1 Role of utility software Year 10 OCR

Utility programs maintain, protect or optimise the system. They are system software but not the OS itself.

OCR names three utilities for exams: encryption, defragmentation, data compression. Schools also teach antivirus, backup and disk cleanup — useful context, but learn the OCR trio first.

Worked example: Windows includes Defrag (HDD), BitLocker (encryption) and supports ZIP (compression) — separate from opening Word.

3.2 Encryption software Year 11 OCR

Encryption utilities scramble data so it cannot be read without the correct key (password or certificate). Protects confidentiality if a laptop is stolen.

Examples: BitLocker (Windows), FileVault (macOS), VeraCrypt. Works with OS security but adds protection even if someone removes the drive.

Worked example: A teacher's USB stick is lost — if encrypted, finder sees random data, not pupil names and grades.

3.3 Defragmentation Year 11 OCR

On a traditional HDD, files can be split into fragments scattered on the disk. The read head moves more, slowing access.

Defragmentation reorganises files into contiguous blocks — can improve speed on HDDs.

SSD note: defragmentation is usually unnecessary and can cause extra wear — OCR exams often contrast HDD vs SSD here.

Worked example: After months of edits, a video project is fragmented — defrag places blocks together; opening the file may load faster on an HDD.

3.4 Data compression Year 10 OCR

Compression utilities (ZIP, RAR, 7-Zip) reduce file size for storage or transmission. Lossless compression for documents restores the exact original — important for coursework and backups.

Worked example: A 400 MB folder of photos zips to 120 MB for email — receiver decompresses to get identical files back.

3.5 Other common utilities (context) Year 10 OCR

Not all are named in OCR 1.5.2, but you may see them in questions or real systems:

  • Antivirus / anti-malware — scans for threats
  • Backup utilities — copy data to another drive or cloud
  • Disk cleanup — remove temporary files
  • Firewall software — filter network traffic (often built into OS)

Worked example: Windows Defender is a utility; it is not the same as the Windows kernel managing memory.
