1 Computational Thinking Year 10

Before you write code, you plan how to solve the problem. Computational thinking is the set of skills that helps you design solutions that a computer (or a person following steps) can carry out. GCSE exams often describe a scenario and ask which skill is being used.

1.1 Abstraction Year 10

Abstraction means focusing on important information and hiding detail that is not needed for the current task. Real systems are complicated; abstraction lets you work with a simpler model.

Example: A school timetable app stores each teacher’s name, room, email, and phone number. When a pupil views “Who teaches me on Monday?”, the app only shows name and room. The email and phone are abstracted away because they are irrelevant to that screen.

In programming, abstraction also means using functions or modules without knowing every line inside them — you trust the interface (what goes in, what comes out).

1.2 Decomposition Year 10

Decomposition means breaking a large problem into smaller sub-problems that are easier to solve, test, and debug. Each sub-task can be given to a different part of the program (or a different person in a team).

Example: “Build a quiz game” could be decomposed into:

  1. Store questions and correct answers
  2. Display one question and read the user’s answer
  3. Check the answer and update the score
  4. Show final results and ask to play again

Examiners may ask you to list sub-tasks in a sensible order — think about what must happen first (e.g. input before processing).

1.3 Pattern recognition Year 10

Pattern recognition means spotting similarities, trends, or rules in problems or data so you can reuse a solution you have seen before.

Example: You notice that every password must have at least one digit. When you write validation code for a new website, you reuse the same “check for a digit” idea — you recognised a pattern from past tasks.

Sequences in exams (2, 4, 8, 16, …) are pattern recognition: the rule is “multiply by 2”, so the next term is 32.

2 Representing Algorithms Year 10

An algorithm is a finite, ordered set of steps that solves a problem. It must be clear enough that someone (or a computer) can follow it without guessing. At GCSE you express algorithms in flowcharts and pseudocode before writing Python.

2.1 Flowcharts Year 10

Flowcharts use standard shapes linked by arrows to show the order of steps. Each shape tells you what kind of step happens — start/stop, input, calculation, or a yes/no choice. You must learn these symbols; exam papers often show or ask you to complete a flowchart.

Standard flowchart symbols

SymbolUsed forTypical GCSE wording
Rounded rectangleStart and end of the algorithmSTART, STOP, END
RectangleA process that changes data“Set total ← 0”, “Calculate average”
DiamondA condition with two exits“Is score ≥ 50?”, label branches Yes / No
ParallelogramInput from or output to the userINPUT age, OUTPUT "Pass"

Worked example: “Is the user an adult?”

This algorithm asks for the user’s age, then prints Adult or Child. Trace it with your finger: only one path runs after the diamond.

Reading the diagram: START → read age → test the condition → exactly one OUTPUT runs → both paths lead to STOP. A rectangle could be added between INPUT and the decision if you had a separate calculation step (e.g. “increment visit count”).

2.2 Pseudocode Year 10

Pseudocode is structured English (or a simple notation) that looks like code but is not tied to one language. Exam boards use keywords such as INPUT, OUTPUT, IF, FOR, WHILE. Assignment is often written with ← (e.g. total ← 0).

Example: Average of two numbers:

INPUT a
INPUT b
average ← (a + b) / 2
OUTPUT average

Indentation inside IF and loops shows which lines belong to which block — same idea as Python.

3 Searching Algorithms Year 10
3.1 Linear search Year 10

Linear search checks each item in turn from the start until it finds the target or runs out of items. It works on sorted or unsorted lists.

Example: List [4, 7, 2, 9], search for 7 → compare 4 (no), 7 (yes) → 2 comparisons. Worst case for n items: n comparisons (target last or missing).

found ← FALSE
FOR i ← 0 TO LEN(list) - 1
    IF list[i] = target THEN
        found ← TRUE
    ENDIF
ENDFOR

Quick Check

Linear search on [5, 3, 8, 1] for 8. How many comparisons until found?

3.2 Binary search Year 11

Binary search only works on a sorted list. Compare the target with the middle item; discard the half where the target cannot be; repeat until found or no items left.

Example: Sorted [2, 5, 9, 14, 18, 23], find 14. Mid = 9 (too small) → search right half [14, 18, 23] → mid 18 (too big) → [14] → found. Far fewer steps than checking every item.

Remember: mid index = (low + high) DIV 2. Update low or high after each comparison.

4 Sorting Algorithms Year 10
4.1 Bubble sort Year 10

Bubble sort repeatedly walks along the list comparing adjacent pairs. If the left item is bigger than the right, swap them. After one full pass, the largest value has “bubbled” to the end. Repeat until a pass completes with no swaps.

Worked example (one pass): [5, 3, 8, 1] → compare 5&3 (swap) → [3, 5, 8, 1] → 5&8 (no swap) → 8&1 (swap) → [3, 5, 1, 8]. Largest value 8 is now at the end.

Quick Check

After one complete pass of bubble sort on [4, 2, 7, 1], which value is definitely in the final position?

4.2 Merge sort Year 11

Merge sort uses divide and conquer: split the list in half until each part has one item, then merge pairs in sorted order.

Example: [38, 27, 43, 3] → split → [38][27][43][3] → merge pairs → [27, 38] and [3, 43] → merge those → [3, 27, 38, 43]. Merging compares the front of each list and takes the smaller.

More efficient than bubble sort for large lists, but needs extra space to hold merged sections — a common discussion point at GCSE.

5 Trace Tables Year 10
5.1 Reading and building trace tables Year 10

A trace table is a dry run of an algorithm: each row shows variable values after a step executes. Columns usually include the step number, each variable, and OUTPUT if something is printed.

Example:

x ← 10
y ← 3
x ← x + y
OUTPUT x
StepxyOUTPUT
1103
2133
313313

Work line by line; only update columns that change on that step.

5.2 Tracing loops and decisions Year 11

For IF, add a row when the condition is tested and record which branch runs. For loops, add a column for the loop counter (i) and a new row each time the loop body runs.

Example:

total ← 0
FOR i ← 1 TO 3
    total ← total + i
ENDFOR
OUTPUT total
StepitotalOUTPUT
start—0
loop 111

loop 336
output—66

Binary search and bubble sort questions often ask for a partial trace — show low, high, mid, or swaps each pass.
