1 CPU & System Architecture Year 10 AQAOCREdexcel

Most general-purpose computers follow the Von Neumann architecture: one shared memory holds both programs (instructions) and data. The CPU fetches instructions and processes data using buses linking memory and I/O devices.

1.1 CPU components Year 10
  • ALU — performs arithmetic (+, −) and logic (comparisons, AND/OR)
  • Control Unit (CU) — coordinates the FDE cycle and signals on buses
  • Registers — tiny, ultra-fast storage inside the CPU (e.g. Program Counter, CIR)
  • Clock — synchronises operations; higher clock speed → more cycles per second
  • Cache — small fast memory between CPU and RAM for frequent data

Example: Adding two numbers — operands move to the ALU; the CU ensures the correct instruction runs at the right time.

1.2 Buses and Von Neumann Year 11

Three buses connect components:

  • Address bus — which memory location to access
  • Data bus — the actual data transferred
  • Control bus — signals (read/write, interrupt, clock)

Because instructions and data share memory, the CPU cannot always fetch both at once — the Von Neumann bottleneck.

2 Fetch–Decode–Execute Cycle Year 10 AQAOCREdexcel
2.1 The three stages Year 10
Fetch — copy the next instruction from memory (address in Program Counter) into the Current Instruction Register (CIR). PC increments.
Decode — the Control Unit works out what the instruction means (e.g. ADD, LOAD, STORE).
Execute — the operation runs (ALU calculates, data moved to/from memory).

The cycle repeats billions of times per second (driven by the clock).

2.2 Worked example — tracing one instruction Year 11

Scenario: PC = 200. Memory address 200 contains ADD 3.

  1. Fetch: Instruction copied to CIR; PC becomes 201.
  2. Decode: CU recognises “add 3 to accumulator”.
  3. Execute: ALU adds 3; result stored in the appropriate register.

Exam questions may ask you to name registers changed or the next PC value.

3.2 Virtual memory Year 11 OCR

When RAM is full, the OS can move less-used pages to secondary storage (hard drive/SSD) and treat that space as extra memory — virtual memory.

Example: You have 8 GB RAM but open apps needing 10 GB. The OS swaps data in and out. The PC still runs, but thrashing (constant swapping) feels very slow.

4 Secondary Storage Year 10 AQAOCREdexcel
4.1 Storage types Year 10
  • HDD — magnetic, moving parts, large capacity, cheaper per GB, slower
  • SSD — flash memory, no moving parts, faster, durable, common in laptops
  • Optical — DVD/Blu-ray; good for distributing read-only media
  • USB / cloud — portable or remote secondary storage

All are non-volatile — data remains without power.

4.2 Choosing storage — worked comparison Year 11 AQA

Example — gaming laptop: Install games on an SSD for fast load times; store old videos on a large HDD if cost matters.

Cloud storage keeps files on remote servers (still magnetic/SSD at the data centre). Advantages: access anywhere, easy sharing. Disadvantages: needs internet, privacy/security concerns, ongoing cost.

Example — backup: Use cloud storage or external USB for off-site copies (not the same as RAM).

Defragmentation helps HDDs, not SSDs (can wear flash unnecessarily).

5 Operating Systems & Software Year 10 AQAOCREdexcel
5.1 Functions of the OS Year 10

The operating system is system software that manages hardware and provides services for applications:

  • Processor management — scheduling which program uses the CPU (multitasking)
  • Memory management — allocating RAM, virtual memory
  • File management — folders, permissions, saving/loading files
  • Security — users, passwords, malware protection
  • Device drivers — communicate with printers, GPUs, etc.

Example: When you open a browser and a music app, the OS time-slices the CPU so both appear to run together.

5.2 Application and utility software Year 10

Application software helps users perform tasks — spreadsheets, games, web browsers.

Utility software maintains or optimises the computer — antivirus, backup tools, disk cleaner, defragmenter (HDD), file compression.

Example: Windows is the OS; Microsoft Excel is an application; a virus scanner is a utility.

6 Embedded Systems & Performance Year 10 AQAOCREdexcel
6.1 Embedded systems Year 10

An embedded system is a computer built into a device to perform one dedicated function. It often has limited RAM/storage, low power use, and real-time requirements.

Examples: washing machine controller, engine management in a car, traffic light sequencer, hospital infusion pump.

Unlike a PC, you cannot usually install any app you want — firmware is specialised.

6.2 CPU performance factors Year 11 AQAOCR
  • Clock speed (GHz) — more cycles per second per core (not the only factor)
  • Cores — more independent processors for parallel tasks
  • Cache size — larger cache can reduce waits for RAM
  • Cooling — heat sinks/fans prevent thermal throttling

Example: Video editing benefits from many cores; a single-threaded game may care more about high clock speed on one core.

6.3 Startup: BIOS/UEFI Year 11

When you press power, firmware (BIOS or UEFI in ROM/flash) runs first. It checks hardware (POST), lets you change basic settings, then boots the OS from a storage device.

Example: “Press F2 for setup” appears before Windows loads — that is firmware, not the OS.
