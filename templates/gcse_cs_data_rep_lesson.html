0 What is data representation? Start here
Everyday idea: When you send a photo on WhatsApp, your phone does not send “colours” and “shapes” in a way humans see them. It sends a long list of numbers. Those numbers are stored as 0s and 1s inside the file. Data representation is the topic about how those 0s and 1s stand for numbers, letters, pixels and sound.

In GCSE you will:

  • convert between normal numbers (denary), binary and hex;
  • add binary numbers (same idea as column addition in maths);
  • work out how big a text file, image or sound clip is;
  • explain compression (ZIP vs JPEG-style).

1 Binary & Denary AQAOCREdexcel
1.1 Why computers use binary
Everyday idea: A light switch is either off or on. Inside a chip, millions of tiny switches work the same way. We call off = 0 and on = 1. That is one bit (binary digit).

Using only 0 and 1 is reliable. Trying to store ten different voltage levels (for denary 0–9) on every wire would cause mistakes when the hardware gets warm or old.

More bits = more combinations, like more digits in a PIN:

  • 1 bit → 2 choices (0 or 1)
  • 4 bits → 16 choices (enough for one hex digit)
  • 8 bits → 256 choices (one byte — often one character in English text)
Exam answer (short): “Computers use binary because electronic circuits represent two stable states (on/off), which maps directly to 0 and 1.”

1.2 Converting denary ↔ binary
Everyday idea — binary → denary: Imagine four switches labelled 8, 4, 2, 1. Flip a switch on and you add that number. Binary 1101 means: 8 on, 4 off, 2 on, 1 on → 8 + 2 + 1 = 13.
Exam method — binary → denary: Write powers of 2 above each bit (…16, 8, 4, 2, 1 from the right). Add the labels where the bit is 1.
Example: 1010 → 8 + 2 = 10.
Denary → binary: Say you want to pay 13p using only 8p, 4p, 2p and 1p coins. You only have one of each, and the vendor is very picky so he asks for the biggest coins first. Take the largest coin that fits: 8p yes, we can use it, 4p yes (the total now is 12), 2p no (the total would be 14 — too much, so we skip it), 1p yes → 8+4+1 = 13. That pattern of “use / skip” is 1101.
Exam method — denary → binary: Repeatedly divide by 2 and write each remainder. Stop when the answer is 0. Read remainders from bottom to top.

13 ÷ 2 → remainder 1
6 ÷ 2 → remainder 0
3 ÷ 2 → remainder 1
1 ÷ 2 → remainder 1
→ 1101
Common mistake: Reading the remainders top-to-bottom. The first remainder you get is the right-hand bit of the answer.

2 Hexadecimal (Base 16) AQAOCREdexcel
2.1 Why hex exists
Everyday idea: A web colour might be written #FF5733 instead of a 24-character binary string. Hex is a shorthand for binary: one hex digit = exactly four bits (a nybble). Two hex digits = 8 bits = one byte.

After 9, hex uses letters: A=10, B=11, … F=15. So FF in one byte means “both nybbles are full” → 255 in denary (the biggest 8-bit unsigned value).

2.2 Converting hex, binary and denary
Everyday idea — hex → denary: Hex 2A is like saying “2 tens and A ones” but in base 16: (2 × 16) + 10 = 42.
Everyday idea — binary ↔ hex: Split the binary into chunks of four bits (from the right). Each chunk is one hex digit — like grouping letters into syllables.
1010 1101 → first chunk 1010 = A, second 1101 = D → hex AD.
Exam method: Pad with leading zeros if needed (110101 → 00110101). Denary → hex: divide by 16, remainders 10–15 become A–F.

3 Binary Arithmetic AQAOCREdexcel
3.1 Binary addition
Everyday idea: You already add in columns in maths. Adding 27 + 15:
  • Units column: 7 + 5 = 12 → write 2, carry 1 to the tens column
  • Tens column: 2 + 1 + carry = 4
  • Answer: 42
Binary addition is the same process. The only difference: each column can only show 0 or 1, so whenever the column sum is 2 or more, you write the remainder (0 or 1) and carry 1 to the next column — exactly like when 7 + 5 became “write 2, carry 1”.

Step 1 — the only “new” column sum: In binary, 1 + 1 = 2. You cannot write 2 in one column, so you write 0 and carry 1 (just like 12 in the units column became “write 2, carry 1”).

Step 2 — practise on small numbers you know: Add 5 + 3 in denary and in binary.

Denary: 5 + 3 = 8

Binary:   101 (5) + 011 (3)
Right column: 1 + 1 = 2 → write 0, carry 1
Middle: 0 + 1 + carry = 0, carry 1
Left: 1 + 0 + carry = 1
Answer: 1000 = 8 ✓

Step 3 — exam-style example: Add 6 + 3 → binary 0110 + 0011.

carry → 1 1 0 (carries into next column) 0 1 1 0 (6) + 0 0 1 1 (3) --------------- 1 0 0 1 (9) Column by column (start on the RIGHT): • 0 + 1 = 1 → write 1, no carry • 1 + 1 = 2 → write 0, carry 1 • 1 + 0 + carry = 2 → write 0, carry 1 • 0 + 0 + carry = 1 → write 1

Check in denary: 6 + 3 = 9, and 1001 = 8 + 1 = 9 ✓

Summary (after you understand the steps):
  • 0 + 0 → write 0, no carry
  • 0 + 1 or 1 + 0 → write 1, no carry
  • 1 + 1 → write 0, carry 1 (and treat “1 + 1 + carry” the same way as in denary)

3.2 Shifts and overflow
Everyday idea — left shift: Writing 37 in denary and shifting digits left (adding a zero) makes 370 — you multiplied by 10. In binary, shifting every bit one place left (and putting 0 in the empty end) doubles the value: 5 → 10 → 20.
Everyday idea — right shift: Shifting 370 right (dropping the last digit) gives 37 — you divided by 10. In binary, shift right once ≈ divide by 2 (whole number, ignore remainder).
Exam method: Left shift by n places (no overflow) → multiply by 2ⁿ. Right shift by 1 (unsigned) → integer divide by 2.
Everyday idea — overflow: A car odometer shows 99999 miles. Add one mile and it shows 00000 — it ran out of digits. An 8-bit register maxes at 255; add 1 and it can “wrap” to 0 because there is nowhere to store 256.

4 Units of Storage AQAOCREdexcel
4.1 Bits, bytes and prefixes
Everyday idea: A text message “Hi” is not stored as ink on paper — the phone stores a short list of numbers for H and i. Each character needs a small chunk of memory; GCSE calls that chunk a byte (8 bits). A photo is thousands of times bigger, so we use kilobytes and megabytes.

Bit = one 0 or 1. Byte = 8 bits. KB in many GCSE questions = 1000 bytes (read the question — sometimes it says 1024).

Watch the capital letter: Mb (megabit) is used for download speed; MB (megabyte) is file size.

4.2 Capacity calculations
Everyday idea: A combination lock with 3 wheels, each 0–9, has 10 × 10 × 10 = 1000 codes. A binary field with 3 bits has 2 × 2 × 2 = 8 patterns (0–7). The biggest code is always one less than the number of patterns: 3 bits → max value 7 (formula: 2ⁿ − 1).
Exam examples:
• 24 bits of data → bytes: 24 ÷ 8 = 3 bytes
• 5 bits unsigned max → 2⁵ − 1 = 31
• 5 KB (if 1 KB = 1000 B) → 5 × 1000 = 5000 bytes

5 Representing Text AQAOCREdexcel
5.1 ASCII and Unicode
Everyday idea: Morse code uses dots and dashes for letters — the receiver looks up each pattern in a table. Computers do the same with numbers: A might be stored as 65, B as 66, and so on. That table is called a character set.

ASCII is the set GCSE questions usually mean. English letters, digits and punctuation each have a code. Often 7 bits per character in theory; many systems store one character in 1 byte (8 bits).

Exam example — file size: Word “Hi” = 2 characters. If 1 byte per character → about 2 bytes (read the question for 7-bit ASCII → 2 × 7 = 14 bits).
Everyday idea — why Unicode: ASCII only has codes for basic English. It has no proper slot for Welsh ŵ, Arabic, Chinese, or emoji 😀. Unicode is a much bigger table so apps worldwide show the right symbol. English codes 0–127 still match ASCII.
AQA note: If the question says 7-bit ASCII, multiply the character count by 7 (bits), not 8.

6 Representing Images AQAOCREdexcel
6.1 Resolution and colour depth
Everyday idea: Zoom into any digital photo and you see tiny coloured squares — pixels, like a mosaic. A bigger mosaic (more pixels across and down) looks sharper: that is resolution (width × height). How many colours each square can be depends on colour depth — more bits per pixel → more possible colours.

Examples: 1 bit per pixel → only black or white (2 colours). 8 bits → 2⁸ = 256 colours. 24 bits → millions of colours (typical phone photo).

6.2 Image file size (uncompressed)
Everyday idea: A 10×10 mosaic has 100 tiles. If each tile needs a 24-digit colour code, you need 100 × 24 = 2400 digits of data. Replace “tiles” with pixels and “digits” with bits — that is the file size formula.
Exam method:
Bits = width × height × colour depth (bits per pixel)
Then ÷ 8 for bytes, ÷ 1 000 000 for decimal MB if asked.

Example: 800 × 600, 24-bit colour
800 × 600 × 24 = 11 520 000 bits
÷ 8 = 1 440 000 bytes ≈ 1.44 MB
Common mistake: Adding width + height + depth. You are counting every pixel’s colour — always multiply all three.

7 Representing Sound AQAOCREdexcel
7.1 Sample rate and bit depth
Everyday idea: Real sound is a smooth wave. A computer cannot store every point on the wave — it takes snapshots (samples) many times per second, like a flip-book animation. Sample rate = how many snapshots per second (CD quality ≈ 44 100 per second). Bit depth = how accurately each snapshot height is stored (more bits → smoother replay, bigger file).

More samples per second or more bits per sample → better quality → larger file. Voice notes on a phone use lower settings than studio music.

7.2 Sound file size (uncompressed)
Everyday idea: A 10-second voice clip with more snapshots per second stores more numbers. Stereo headphones need a height value for left and right each time — twice the data of mono.
Exam method:
Bits = sample rate × bit depth × duration (seconds) × channels
(channels = 1 for mono, 2 for stereo)

Example: 44 100 Hz, 16-bit, 10 s, mono
44 100 × 16 × 10 = 7 056 000 bits → ÷ 8 = 882 000 bytes
Same clip in stereo → × 2 = 1 764 000 bytes

8 Compression AQAOCREdexcel
8.1 Lossy and lossless compression
Everyday idea — lossless: Instead of writing “WWWWWWWWWW” on a form, you write “10 × W”. Anyone can expand it back to the exact original. That is run-length encoding (RLE) — a lossless trick exams love.
Everyday idea — lossy: Summarising a long film plot loses detail, but the summary is shorter. JPEG photos and MP3 music throw away data your eyes/ears rarely notice. The file shrinks a lot, but you cannot recover every original bit.
Exam choices:
• Python homework, ZIP, exact graphics → lossless (must get every bit back)
• Streaming music / social photos → often lossy (smaller, “good enough” quality)

Data representation — exam checklist

Conversions

  • Denary → bin: divide by 2, remainders up
  • Bin → denary: sum powers of 2 (switch/coin idea)
  • Bin ↔ hex: nybbles of 4 bits

Formulas

  • Max n-bit unsigned: 2ⁿ − 1
  • Image bits: W × H × depth
  • Sound bits: rate × depth × s × ch

Binary add

  • Same as column add in maths
  • 1 + 1 → write 0, carry 1
  • Left shift → ×2
