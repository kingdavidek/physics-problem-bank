0 What is Cyber Security? Year 10 AQAOCREdexcel
0.1 Definition and aims Year 10 AQAOCREdexcel

Cyber security is the set of processes, practices and technologies used to protect networks, computers, programs and data from attack, damage or unauthorised access.

Exams often link three goals (useful across all boards):

  • Confidentiality — only authorised people see the data
  • Integrity — data is accurate and not tampered with
  • Availability — systems and data are there when needed (DoS attacks target this)
Defence in depth — no single control is perfect; layers (physical, network, software, people, policies) work together.

Worked example: A school stores pupil marks on a server. Cyber security means: locked server room (physical), firewall (network), patched OS (software), strong staff passwords (people), and an AUP pupils sign (policy).

1 Threats to Systems Year 10 AQAOCREdexcel
1.1 Malware Year 10

Malware (malicious software) is hostile or intrusive software that harms systems or steals data. Exam boards require virus, trojan and spyware; Edexcel also names worms, ransomware and key loggers.

TypeWhat it does
VirusAttaches to files; spreads when infected file is opened
WormSpreads across networks without user opening a file
TrojanDisguised as legitimate software; user installs it
SpywareRecords activity secretly (keystrokes, screens)
Key loggerCaptures keyboard input to steal passwords
RansomwareEncrypts or locks files; demands payment

Protection: anti-malware with up-to-date definitions, do not run unknown attachments, least-privilege accounts, backups.

Example: A pupil opens “free_game.exe” — a Trojan installs spyware that logs keystrokes.

1.2 Social engineering Year 10

Social engineering manipulates people into breaking security — the “human weak point”. People are often the weakest link in a system.

  • Phishing — fake emails, texts (smishing) or calls (vishing) to steal credentials
  • Blagging / pretexting — invented story to gain trust (“I’m from IT, read me your password”)
  • Shoulder surfing / shouldering — watching someone enter a PIN or password
Red flags: urgent threats, spelling errors, odd sender addresses, links that do not match the real domain.

Protection: security awareness training, verify callers independently, screen shields at ATMs, never share passwords, report suspicious messages.

Worked example: Email says “Your school password expires today — click here.” Link goes to sch00l-login.evil.com — phishing.

1.3 Configuration & physical threats Year 10 AQAEdexcel

Not every threat is malware. You also need to know:

  • Pharming — redirects traffic to a fake website (often via poisoned DNS or hosts file), even if the user types the correct URL
  • Weak and default passwords — e.g. admin/admin on a router; easy for brute force
  • Misconfigured access rights — pupils given admin rights; shared folders open to everyone
  • Removable media — infected USB sticks plug into school PCs; data copied off on unauthorised USB
  • Unpatched / outdated software — known flaws attackers exploit; includes out-of-date anti-malware

Pharming vs phishing: phishing tricks the user into clicking a bad link; pharming misdirects traffic so the address bar looks correct but the site is fake.

Example: A teacher leaves a laptop unlocked in a corridor — insider/careless threat; pupil copies files to a USB (removable media risk).

2 Network & Application Attacks Year 10–11 OCREdexcel
2.1 DoS, brute force and data interception Year 10 OCR

Denial of Service (DoS) — floods a server so legitimate users cannot connect (attacks availability). DDoS uses many machines (botnet).

Brute force — tries many passwords until one works. Slowed by lockouts, CAPTCHA, rate limiting, and strong passwords.

Data interception and theft — capturing packets on unencrypted WiFi, stealing laptops, or MITM on weak links.

Example: Revision site goes offline during exams due to junk requests — DoS.

3.2 Passwords, 2FA, CAPTCHA and email verification Year 10 AQAOCR

Password systems: long, unique passwords; passphrases; password managers; never reuse across sites; change defaults on routers.

2FA / MFA — two different factor types: know (password) + have (phone code) or are (fingerprint).

CAPTCHA (or similar) — proves a human is logging in; slows automated brute force and bots.

Email confirmation — link sent to registered email to verify identity when signing up or resetting password.

Worked example: School email: password + approve notification on phone app — 2FA.

3.5 Backups and recovery Year 11 EdexcelOCR

Full backup — copies everything (slow, simple restore). Incremental — only changes since last backup (faster, longer restore chain).

3-2-1 rule: 3 copies, 2 media types, 1 off-site. Test restores regularly.

Example: Ransomware on a file server — disconnect network, restore from clean off-site backup, investigate breach.

5.3 GDPR, insider threats and incident response Year 11 AQAOCREdexcel

UK GDPR protects personal data. Schools must keep pupil data secure, use it lawfully, minimise what they hold, and respect rights (e.g. access). The ICO regulates this in the UK.

Insider threat — risk from staff or careless users (shared passwords, unlocked laptops, wrong email attachment).

Incident response (e.g. ransomware): isolate affected systems → report to leadership / ICO if personal data breached → restore from clean backups → review and improve defences.

Worked example: Ransomware on file server — disconnect VLAN, do not pay immediately, restore from off-site backup, log incident for governors.
