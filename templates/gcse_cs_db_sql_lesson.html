1 Databases & the Relational Model Year 10 AQAOCR
1.1 What is a database? Year 10 AQAOCR

A database is an organised store of data that can be searched, added to, updated and reported on electronically. Examples: school pupil records, a shop stock system, a library catalogue.

A relational database stores data in related tables (like multiple linked spreadsheets). Each table holds one type of entity (pupils, classes, grades).

Worked example: Instead of one huge spreadsheet with repeated class teacher names on every pupil row, you store teachers once in Class and link pupils with a ClassID.

1.3 Redundancy and consistency Year 10 AQA

Data redundancy — storing the same fact twice (e.g. teacher name on every pupil row).

Data inconsistency — copies disagree (e.g. "Mr Jones" on one row, "Mr Jons" on another).

Relational design reduces redundancy by storing each fact once and linking tables with keys — improving consistency.

Worked example: Class teacher changes to "Ms Lee". Update one row in Class — every pupil linked via ClassID sees the correct teacher in reports.

2.2 WHERE — filtering rows Year 10 AQAOCR

WHERE filters which records are returned. Text values use quotes.

SELECT Surname FROM Pupil WHERE YearGroup = 11;
SELECT * FROM Class WHERE ClassName = '10A';

Worked example: Pupils in class 10A (ClassID stored as number 3):

SELECT FirstName, Surname FROM Pupil WHERE ClassID = 3;

3 Linking Two Tables Year 11 AQA
3.1 Queries across Pupil and Class Year 11 AQA

AQA exams may use up to two tables in one query. Link them by matching foreign key to primary key in WHERE:

SELECT Pupil.Surname, Class.ClassName FROM Pupil, Class WHERE Pupil.ClassID = Class.ClassID;

Use Table.Field when column names appear in more than one table.

Worked example: Teacher for each pupil:

SELECT Pupil.FirstName, Class.Teacher FROM Pupil, Class WHERE Pupil.ClassID = Class.ClassID;

5 Designing Databases & Staying Safe Year 11 OCR
5.1 Planning tables (ERD) Year 11 OCR

Before writing SQL, plan entities (tables), attributes (fields), and relationships (keys). An entity-relationship diagram (ERD) shows boxes for tables and lines for links.

Worked example — library system:

  • Member(MemberID PK, Name)
  • Book(BookID PK, Title, Author)
  • Loan(LoanID PK, MemberID FK, BookID FK, DueDate)

Choose sensible data types: INTEGER for IDs, TEXT for names, DATE for DueDate.

5.2 Validation and SQL injection Year 11 OCR

Input validation checks data before it is stored (e.g. year group must be 7–13, name not empty).

SQL injection — attackers insert malicious SQL into input fields. Defences: validate input, use parameterised queries (do not glue user text straight into SQL strings).

Example attack string: ' OR '1'='1 in a login box could trick a poorly written query.
